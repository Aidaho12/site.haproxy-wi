The checker service consists of two parts:

  • Master checker.
  • Worker checker.
The "Master checker" runs the "Worker checker". The "Master checker" should be running always.

When you want to start monitoring HAProxy service the The "Master checker" will start local worker for every HAProxy service. The "Worker checker" will check every backend server and HAProxy status remotly by 1999 port(by default). If any backend lost connect with the HAProxy service or the HAProxy service is fallen, the checker will send message via Telegram about it If you want to use and if you have install HAProxy service manually read how you should set HAProxy service.

The "Master checker" and "Worker checker" Logs
